# qPan

> 全自动 QQ 群文件网盘管理机器人插件

基于 NoneBot2 的 QQ 群文件网盘管理插件，支持多群联动、智能空间管理、文件自动永久保存等功能。

## ✨ 功能特性

- 🤖 **智能监控**：自动监听群文件上传事件
- 💾 **自动永久保存**：上传后自动设置文件为永久保存（带重试机制）
- 📦 **多群联动**：支持跨多个群盘统一管理文件
- 🔄 **智能转移**：空间不足时自动将文件转移到有剩余空间的群盘
- 📊 **空间统计**：实时查看所有群盘的使用情况
- 🔍 **文件搜索**：快速搜索文件名关键词
- 📑 **分页浏览**：支持分页查看文件列表，可筛选永久/非永久文件
- 🌐 **HTTP 直下**：使用 httpx 流式下载，避免大文件超时

## 📋 环境要求

- Python >= 3.10
- NoneBot2 >= 2.4.4
- OneBot V11 协议端（如 go-cqhttp、Lagrange）

## 🚀 快速开始

### 安装依赖

```bash
# 克隆项目
git clone <your-repo-url>
cd qPan

# 安装依赖
pip install -e .
```

### 配置 OneBot 协议端

确保你的 OneBot 实现端（如 Lagrange）已正确配置并连接到 NoneBot2。

### 群名称配置

插件通过**群名称包含 "qpan"** 来识别管理的群盘，请将需要管理的 QQ 群名称改为包含 `qpan` 的形式，例如：
- `qpan1号盘`
- `我的qpan存储`
- `qpan-备份群`

### 启动机器人

```bash
# Windows
qstart.bat

# Linux/macOS
nb run
```

## 📖 使用说明

### 命令列表

| 命令 | 别名 | 参数 | 说明 |
|------|------|------|------|
| `qpan help` | `群盘 帮助` | - | 查看帮助信息 |
| `qpan info` | `群盘 总盘` | - | 查看所有群盘空间使用情况 |
| `qpan list [页码] [筛选]` | `群盘 列表` | 页码(可选), 筛选(0/1) | 查看文件列表，0=非永久，1=永久 |
| `qpan search <关键词>` | `群盘 搜索` | 关键词 | 搜索包含关键词的文件 |

### 使用示例

```
# 查看所有群盘总空间
qpan info

# 查看第 1 页所有文件
qpan list

# 查看第 2 页所有文件
qpan list 2

# 查看第 1 页，只显示永久文件
qpan list 1 1

# 查看第 1 页，只显示非永久文件
qpan list 1 0

# 搜索文件名包含 "报告" 的文件
qpan search 报告
```

### 自动化功能

#### 文件上传监听
当群成员上传文件时，机器人会：
1. 自动检测上传事件
2. 计算当前群盘空间使用率
3. 判断是否有足够空间存储新文件
4. 发送空间使用报告

#### 自动永久保存
如果群盘有足够空间：
- 自动将新上传的文件设置为永久保存
- 使用延迟重试机制（2/4/6秒）避免 API 限流
- 成功或失败后发送通知消息

#### 智能空间转移
如果群盘空间不足：
- 后台任务自动下载文件到本地 `downloads/` 目录
- 查找有足够空间的其他 qpan 群
- 自动上传文件到目标群盘
- 转移完成后发送结果通知

## ⚙️ 配置说明

### 项目结构

```
qPan/
├── src/
│   └── plugins/
│       ├── qpan.py          # 主插件文件
│       └── __pycache__/
├── downloads/                # 文件下载临时目录（自动创建）
├── pyproject.toml           # 项目配置
├── qstart.bat               # Windows 启动脚本
└── README.md
```

### 自定义配置

可以在 `src/plugins/qpan.py` 中修改以下配置：

```python
# 每页显示文件数量
max_files_per_page = 10

# 文件下载超时设置（秒）
timeout=httpx.Timeout(connect=30, read=120, write=30, pool=30)

# 永久保存重试次数
max_retries = 3
```

- **asyncio**：Python 异步编程支持

## ⚠️ 注意事项

1. **群名称要求**：只有群名称包含 "qpan" 的群才会被管理
2. **权限要求**：机器人需要有群文件上传/下载/管理权限
3. **空间预留**：建议为每个群盘预留一定空间，避免完全填满
4. **大文件处理**：大文件下载转移可能需要较长时间，过程在后台执行
5. **API 限制**：QQ 官方对文件操作有频率限制，建议合理使用
